### 2. 医保低付（金额差异）

**定义**：HIS申报金额 > 医保实际支付金额，存在扣款。

**示例记录**：

- 扣款幅度：17%~40%

**对齐键**：`patient_id + visit_date + item_code`

## 数据构造指南

### 构造完全匹配数据

1. 确保 `patient_id`、`visit_date`、`item_code` 三键完全一致
2. HIS `amount` = 医保 `pay_amount`
3. 医保 `reject_reason` 为空

### 构造金额差异数据

1. 三键对齐后，`his.amount > insurance.pay_amount`
2. 医保 `reject_reason` 填写扣款原因（如：超出限制性规定、频次不符等）
3. 可模拟不同扣款幅度：10%、20%、30%、50%

### 构造单边缺失数据

1. **HIS多**：医保 `settlement_no` 中无对应记录
2. **医保多**：HIS `settlement_no` 中无对应记录
3. 可模拟传输延迟丢包场景

### 构造语义差异数据

1. 金额差异的拒付原因需包含**语义信息**（如：检查指征、频次限制、数量不符）
2. 避免仅填"系统误差"等无意义原因
3. 可让LLM根据原因文本进行语义聚类

## 验证流程

运行以下命令验证示例数据：

```bash
# 分析差异
./bin/reconcile analyze data/his_sample.csv data/insurance_sample.csv --output output/test_result.json

# 生成HTML报告
./bin/reconcile report output/test_result.json --type detail --format html --output output/test_report.html

# 查询差异
./bin/reconcile query output/test_result.json --cluster amount_difference
```

**预期结果**：
- 完全匹配：9条
- 金额差异（医保低付）：6条
- HIS单边缺失：11条
- 医保单边缺失：2条
- 差异总金额：约 217.00元

## 数据脱敏说明

示例数据均为**模拟数据**，患者ID、项目编码、金额等均为虚构，用于测试和演示目的。

如需构造真实场景数据：
1. 将真实患者ID替换为脱敏后的伪ID（如：真实身份证号 → P+序号）
2. 项目编码需使用真实的医保目录编码（如：国家医保编码）
3. 金额可保留真实口径
4. 拒付原因可保留真实描述
